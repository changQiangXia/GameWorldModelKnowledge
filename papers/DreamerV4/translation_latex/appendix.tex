\section{数据集}

\paragraph{Minecraft VPT}

我们使用 OpenAI VPT 的承包者游戏数据集\citep{vpt}，并合并可用的 6--10 号子集，总计 2541 小时游戏记录。
数据集划分为 90\% 训练集和 10\% 评估集，并保证两个划分不共享同一个底层 5 分钟录制片段。
键盘动作编码为二值变量向量；鼠标动作按照 VPT 的做法使用 $\mu$-law 编码，每个坐标离散为 11 个桶，并枚举所有 $11 \times 11 = 121$ 种组合，得到一个分类变量。
图像分辨率为 $360 \times 640$，帧率为 20 FPS。
我们将视频帧零填充到 $384 \times 640$，再以 $16 \times 16$ 的 patch 大小切分为 960 个 token。
分词器的瓶颈表示大小为 $(N_\mathrm{b} = 512) \times (D_\mathrm{b} = 16)$，我们将其重排为 $(N_\mathrm{z} = 256) \times 32$ 后输入动力学模型。
训练动力学模型时，空间 token 数为 $N_\mathrm{z} = 256$，上下文长度为 $C = 192$，批次长度为 $T_1 = 64$ 和 $T_2 = 256$。

\paragraph{Minecraft Overworld 与 Nether 划分}

为研究 \method 中动作条件的分布外泛化，我们将 Minecraft 数据集仔细划分为 Overworld 视频和 Nether 维度视频。
End 维度也被并入 Nether 部分。
与 Overworld 相比，Nether 和 End 在视觉外观、方块和地形形状上都有明显差异。
Overworld 包含森林、沙漠、海洋等自然景观；Nether 以红色方块和熔岩为主，End 则类似太空区域。
划分数据集时，我们需要避免玩家进入 Nether/End 后把其中的方块带回 Overworld，造成数据泄漏。
因此，我们排除了包含长时间自由游玩的 VPT 6 和 7 子集。
随后，根据数据集提供的物品事件，将剩余数据中的每个 5 分钟录制片段分配到 Overworld 或 Nether/End 部分。
只要视频中出现过与 Nether/End 物品的交互，就将该视频归入 Nether/End 划分。
这样可以保证 Overworld 划分不包含 Nether/End 回合；Nether/End 划分偶尔会包含少量 Overworld 回合，但实践中很少发生。
我们人工检查了按该策略得到的 Overworld 划分，没有发现 Nether/End 轨迹。

\paragraph{SOAR Robotics}

SOAR 数据集\citep{soar} 包含遥操作示范和强化学习策略的在线轨迹，因此同时覆盖成功与失败样本。
我们将数据集划分为 90\% 训练集和 10\% 评估集。
该数据集共有 180 小时视频，并带有 7 维相对末端执行器动作。
图像分辨率为 $256 \times 256$，帧率为 5 FPS。
我们以 $16 \times 16$ 的 patch 大小将图像切分为 256 个 token。
训练动力学模型时，空间 token 数为 $N_\mathrm{z} = 512$，上下文长度为 $C = 96$，批次长度为 $T_1 = 32$ 和 $T_2 = 128$。

\paragraph{Epic Kitchens}

Epic Kitchens 100 数据集\citep{epickitchens} 包含来自 45 个厨房的人类第一视角视频，总时长为 100 小时。
测试集包含在相同厨房中执行的不同任务。
我们使用 $256 \times 256$ 分辨率、10 FPS 的数据。
图像以 $16 \times 16$ 的 patch 大小切分为 256 个 token。
训练动力学模型时，空间 token 数为 $N_\mathrm{z} = 512$，上下文长度为 $C = 96$，批次长度为 $T_1 = 32$ 和 $T_2 = 128$。

\section{Dreamer 前代方法}
\vspace*{-1ex}
Dreamer~3\citep{dreamerv3} 通过在线交互从零开始学会在 Minecraft 中获得钻石。
它的输入包括低分辨率图像和物品栏状态，输出为鼠标、键盘以及抽象合成动作。
Dreamer~3 使用循环状态空间模型（RSSM）\citep{hafner2018planet} 作为世界模型，该模型基于循环神经网络和变分目标。
这种方法得到的世界模型较轻量，推理效率很高，但难以扩展到多样化数据分布。
相比之下，\method 完全从离线数据中学习在 Minecraft 中获得钻石。
它只接收高分辨率图像作为输入，输出为底层鼠标和键盘动作。
\method 使用基于高效 Transformer 架构和 shortcut forcing 目标的可扩展世界模型，因此可以适应包含大量细节的多样化数据分布。
Dreamer~3 在想象训练中使用回报归一化和熵正则项，而 \method 使用带行为克隆先验 KL 项的 PMPO，不需要归一化。
