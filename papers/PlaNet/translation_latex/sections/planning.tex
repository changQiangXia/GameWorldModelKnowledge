\section{潜在空间规划}
\label{sec:planning}

\input{algorithms/agent}

为了通过规划解决未知环境，智能体需要从经验中学习环境动力学。\method 采用迭代流程：先用规划收集数据，再用收集到的数据训练动力学模型。本节先给出环境记号，并说明基于模型智能体的整体实现。这里暂且假设已有学习得到的动力学模型；模型设计和训练目标见\cref{sec:model}。

\paragraph{问题设置}

单帧图像通常不能完全揭示环境状态，因此我们将问题建模为部分可观察马尔可夫决策过程（POMDP）。记离散时间步为~$t$，隐藏状态为$s_t$，图像观测为$o_t$，连续动作向量为$a_t$，标量奖励为$r_t$，它们服从如下随机动力学：
\begin{gather}
\begin{aligned}
\makebox[12em][l]{Transition function:} && s_t &\sim\pr[\mathrm{p}](s_t|s_{t-1},a_{t-1}) \\
\makebox[12em][l]{Observation function:} && o_t &\sim\pr[\mathrm{p}](o_t|s_t) \\
\makebox[12em][l]{Reward function:} && r_t &\sim\pr[\mathrm{p}](r_t|s_t) \\
\makebox[12em][l]{策略：} && a_t &\sim\pr[\mathrm{p}](a_t|o_{\leq t},a_{<t}),
\label{eq:pomdp}
\end{aligned}
\raisetag{2.4\baselineskip}
\end{gather}% Intentional empty line to allow column break.

不失一般性，我们假设初始状态$s_0$固定。目标是实现一个策略$\pr[\mathrm{p}](a_t|o_{\leq t},a_{<t})$，使预期累计奖励$\E[\big]{\mathrm{p}}{\sum_{t=1}^T r_t}$最大，其中期望同时取自环境分布和策略分布。

\paragraph{基于模型的规划}

\method 从已有回合中学习转移模型$\p(s_t|s_{t-1},a_{t-1})$、观测模型$\p(o_t|s_t)$和奖励模型$\p(r_t|s_t)$（这里用斜体字母表示模型分布，用直立字母表示真实动力学）。观测模型提供丰富的训练信号，但不参与规划。我们还学习编码器$\q(s_t|o_{\leq t},a_{<t})$，通过过滤从历史中推断当前隐藏状态的近似信念。基于这些组成部分，我们将策略实现为搜索未来最优动作序列的规划算法。模型预测控制\citep[MPC;][]{richards2005mpc}使智能体能够根据新的观测调整计划，因此每个时间步都会重新规划。与无模型和混合强化学习算法不同，我们不使用策略网络或价值网络。

\paragraph{经验收集}

由于智能体一开始可能无法访问环境的所有区域，我们需要不断收集新经验并改进动力学模型。如\cref{alg:agent}所示，我们从$S$个由随机动作收集的种子回合开始训练模型，之后每经过$C$个更新步骤就向数据集中加入一个新回合。收集回合时，我们在动作上加入小幅高斯探索噪声。为缩短规划视野，并向模型提供更清晰的学习信号，我们像强化学习中的常见做法一样，将每个动作重复$R$次\citep{mnih2015dqn,mnih2016a3c}。

\paragraph{规划算法}

我们使用交叉熵方法\citep[CEM;][]{rubinstein1997cem,chua2018pets}在学习模型下搜索最优动作序列，流程见\cref{alg:planner}。选择该算法是因为它足够稳健，并且在使用真实动力学规划时能解决本文考虑的所有任务。CEM 是一种基于种群的优化算法，通过估计动作序列上的分布来最大化目标。附录中的\cref{alg:planner}给出了细节：我们初始化一个随时间变化的对角高斯信念，用来表示最优动作序列$a_{t:t+H}\sim\Normal(\mu_{t:t+H},\sigma^2_{t:t+H}\mathbb{I})$，其中$t$为智能体当前时间步，$H$为规划视野长度。从零均值、单位方差开始，我们反复采样$J$个候选动作序列，在模型下评估它们，并用排名前$K$的序列重新拟合该信念。经过$I$次迭代后，规划器返回当前时间步信念的均值$\mu_t$。需要注意的是，在接收下一次观测后，动作序列的信念会重新从零均值和单位方差开始，以避免陷入局部最优。

为了在学习模型下评估候选动作序列，我们从当前状态信念出发采样一条状态轨迹，并累加沿该序列预测的平均奖励。由于优化器本身基于种群，我们发现每个动作序列只考虑一条轨迹已经足够，从而可以把计算预算用于评估更多不同的序列。奖励被建模为潜在状态的函数，因此规划器可以完全在潜在空间中运行，无需生成图像；这使得大批量动作序列的评估速度更快。下一节介绍规划器所使用的潜在动力学模型。
